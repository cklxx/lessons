## 使用约定（读之前先同意）

1. 无测试不上线：每次功能变更必须提供可复现的验证方式（自动化优先）。
2. 先写门槛再写结论：指标口径、失败判定与回滚动作必须先于“优化/更好”。
3. 默认回滚：发布失败或指标异常时的第一响应是回滚，而不是在线修补。
4. 文档即代码：文档必须能落成补丁，且与代码变更保持同步。
