# [60] LangGraph：把 Agent 变成可控的工作流

- 资料类型：官方文档
- 原始来源：https://docs.langchain.com/oss/python/langgraph/overview/
- 对应章节：第 10 章（Agent：工作流）

## 一句话
别让 Agent 变成一条不可控的长链条，把它画成图，每个节点都有边界与回退。

## 你该从文档带走什么
- **状态机思维**：节点、边、条件分支、终止条件，能把会做变成可控地做。
- **可测试性**：每个节点可以单测/回归；整个图可以做端到端用例。

## 在本书里怎么用
- 第 7 章写作上优先强调：输入/输出/失败回退/审计日志，而不是框架 API 细节。
- 把工具调用当作节点边界：权限、幂等、速率限制、重试策略都写在边界里。

## 常见误用
- 让 Agent 自己决定什么时候结束，没有硬性停止条件与超时策略，最后变成无限循环或成本爆炸。
