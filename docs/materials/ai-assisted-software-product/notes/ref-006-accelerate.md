# [6] Accelerate：交付表现与门禁怎么量化

- 资料类型：工程管理/交付科学（书）
- 原始来源：https://itrevolution.com/product/accelerate/
- 对应章节：第 4/12 章（交付工作流、质量门禁、迭代节奏）

## 一句话
你不是缺“更努力”，你是缺“可量化的交付系统”。

## 你该从这本书带走什么
- **DORA 指标不是 KPI**：它是系统健康度的仪表盘（用来发现瓶颈，不是用来压人）。
- **门禁要服务结果**：高质量团队不是“更多流程”，而是“更短反馈 + 更稳定回滚”。
- **文化与技术是同一件事**：自动化程度、协作方式、故障复盘都会反映在交付表现上。

## 在本书里怎么用
- 给每章的“验证命令/失败判定”补一个“指标落点”：对应质量、交付频率、回滚速度之一。
- 把“看起来更好”改成“指标更好”：没有对比表就不声称优化。

## 常见误用
- 把 DORA 当成个人绩效，而不是系统指标，导致数据失真与隐瞒问题。
- 用“提速”压过“稳定”，最终把故障成本转嫁到线上与用户。
