# [17] WCAG 2.2：可访问性底线与可验收检查项

- 资料类型：标准/规范
- 原始来源：https://www.w3.org/TR/WCAG22/
- 对应章节：第 6 章（UI 设计）、第 8 章（前端实现）

## 一句话
可访问性不是为少数人做的高级选项，而是能不能所有人都用得了的底线。

## 你该从 WCAG 带走什么
- **可访问性可验收**：不是审美争论，而是检查项（对比度、键盘可达、可感知提示等）。
- **看起来能点不等于能用**：键盘顺序、焦点可见、错误提示可读才是关键。

## 在本书里怎么用
- 在示例（可复制）里把 a11y 变成验证命令（例如 axe 扫描/Playwright + a11y）。
- 在常见陷阱里把上线后才发现不可用写成可复现失败样本（换长文案/只用键盘操作）。

## 常见误用
- 只在上线前做一次性检查，缺少回归基线，导致每次改 UI 都退化。
- 只管通过规则，不管用户能完成任务（建议把关键任务写成可访问的交互回归用例）。
