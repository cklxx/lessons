# [29] ReAct：Agent 的推理—行动循环

- 资料类型：论文
- 原始来源：https://arxiv.org/abs/2210.03629
- 对应章节：第 7 章（Agent）

## 一句话
让模型“边想边做”：每一步都把行动结果（Observation）写回上下文，再决定下一步。

## 你该从 ReAct 带走什么
- **链路可观测**：Thought/Action/Observation 的结构能让调试与审计更容易。
- **工具调用要有边界**：只给必要工具、必要权限，否则 Agent 会把“会做”变成“乱做”。

## 在本书里怎么用
- 把 Agent 写成“可回归的工作流”：每个节点有输入/输出/失败回退。
- 把工具调用写成白名单 + 幂等策略 + 速率限制（避免放大错误）。

## 常见误用
- 让模型自由发挥“自己决定要不要调用工具”，但缺少可回滚与审计日志。
