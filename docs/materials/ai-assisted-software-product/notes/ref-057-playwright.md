# [57] Playwright：端到端测试的最后一道门禁

- 资料类型：官方文档/工具
- 原始来源：https://playwright.dev/
- 对应章节：第 7 章（工程化）、第 8 章（前端实现）

## 一句话
单测保证局部正确，E2E 保证用户真能走完流程；Playwright 更适合把关键链路做成自动化门禁。

## 你该从 Playwright 带走什么
- **稳定性与可调试**：回放/trace 能让 flaky 更容易定位。
- **多浏览器覆盖**：减少在我电脑上可以的尴尬。
- **适合关键链路**：注册/支付/订阅/导出等，少而精，能显著降低线上风险。

## 在本书里怎么用
- 第 3 章用它跑 a11y/交互回归；第 5 章用它跑关键业务链路回归。
- 和示例（可复制）绑定：给出验证命令与失败判定，读者照着做就能验收。

## 常见误用
- 把全站所有页面都做成 E2E，导致跑一次要半小时，最后没人跑。
- 选择器不稳定（动态 id），导致页面小改测试就全崩。
