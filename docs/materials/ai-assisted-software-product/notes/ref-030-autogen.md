# [30] AutoGen：多智能体协作不是为了热闹

- 资料类型：论文/框架
- 原始来源：https://arxiv.org/abs/2308.08155
- 对应章节：第 10 章（Agent）

## 一句话
多智能体的价值不是能聊，而是把任务拆成角色分工，让每一步都有边界、有验收、有回退。

## 你该从 AutoGen 带走什么
- **角色分工能降低认知负担**：把规划/实现/审查/测试拆开，比让一个模型全干更稳。
- **协作要可控**：多轮对话必须有停止条件、预算上限与失败回退，否则只会越聊越贵。
- **把协作产物落盘**：日志、决策、工具调用都要可追溯，才能复盘与回归。

## 在本书里怎么用
- 第 7 章用它作为多角色协作的参考实现，但正文强调范式（分工/门禁/审计），不绑定具体框架。
- 写清楚什么时候不需要多智能体（线性任务、明确规则任务），避免过度工程。

## 常见误用
- 简单任务也硬上多智能体，成本/延迟飙升，最后还不如单体链式调用。
- 没有仲裁/停止条件，导致无限循环或回声室。
