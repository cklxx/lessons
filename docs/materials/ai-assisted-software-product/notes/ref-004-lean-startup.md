# [4] 精益创业（The Lean Startup）：最小实验与证据门槛

- 资料类型：商业/产品方法论（书）
- 原始来源：https://theleanstartup.com/
- 对应章节：第 1–2 章（方法论与需求挖掘）

## 一句话
别急着做功能，先做可证伪的实验，用数据决定继续/停止。

## 你该从这本书带走什么
- **Build–Measure–Learn 不是口号**：把一次迭代定义成一个假设 + 一个可观察指标 + 一个最小实验。
- **MVP 的重点不是小**：而是能在最短时间里验证关键风险（需求/渠道/付费意愿/留存）。
- **可执行的停止条件**：没有门槛/阈值的实验等于没有实验，只是在做产品日记。

## 在本书里怎么用（可直接改写成正文动作）
- 把想做什么改写成要验证什么：每个需求点配一条可证伪假设（例如7 天内新增留资 ≥ X）。
- 把收集反馈改写成收集反例：至少设计 1 个会让你放弃方向的观察结果。
- 把做 MVP改成做最小实验装置：能采集数据、能复盘、能回滚。

## 常见误用（写作/落地都容易踩）
- 把 MVP 当成残缺版产品交付给用户，但没有指标与对照组。
- 用用户说喜欢替代用户行为改变（点击、留资、付费、留存）。
