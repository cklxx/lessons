# [27] BM25：关键词检索的底盘（以及它为什么仍然重要）

- 资料类型：综述/方法论
- 原始来源：https://www.nowpublishers.com/article/Details/INR-019
- 对应章节：第 6 章（RAG）

## 一句话
向量检索擅长“意思像不像”，BM25 擅长“关键词对不对”；RAG 里两者最好是搭配着用。

## 你该从 BM25 带走什么
- **关键词就是生产系统的现实**：产品型号、错误码、专有名词、缩写，往往靠关键词更稳。
- **混合检索不是玄学**：把“词面匹配”与“语义匹配”各自的优势拼起来，召回更稳。
- **可解释性更好**：BM25 的匹配理由更容易做抽查与调参记录。

## 在本书里怎么用
- 第 6 章写“混合检索（BM25 + 向量）”时，把 BM25 当成默认配置之一，而不是可选项。
- 把“什么时候 BM25 会更好”写清楚（例如：查错误码、查 API 名、查产品型号）。

## 常见误用
- 只用向量检索，导致“精确词”场景召回崩溃（看起来相关、但就是找不到那条关键证据）。
- BM25 参数不随语料规模/噪声调整，导致检索结果被高频噪声词淹没。
