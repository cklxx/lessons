# [56] Storybook Test Runner：组件级回归比页面级更省钱

- 资料类型：官方文档/工具
- 原始来源：https://storybook.js.org/docs/writing-tests/integrations/test-runner
- 对应章节：第 3 章（UI/UX）、第 4 章（工作流与回归）

## 一句话
先把组件当成“最小可测单元”，再做整页回归；否则每次改 UI 都要付出更高的回归成本。

## 你该从 Test Runner 带走什么
- **组件隔离测试**：把状态（默认/加载/错误/禁用）固定下来，回归更稳。
- **可做交互与视觉回归**：对 AI 生成的组件尤其有用（它最容易漏状态与可访问性）。
- **回归快**：比端到端跑全站更快更便宜，适合高频变更。

## 在本书里怎么用
- 第 3 章把“至少一个组件可回归”的门槛落在 Storybook 上。
- 第 4 章把它作为“左移回归”的例子：越早发现退化越便宜。

## 常见误用
- 让组件测试承担过多业务逻辑，导致用例脆弱、维护成本上升。
- 缺少稳定的状态构造（mock/fixtures），导致测试 flaky。
