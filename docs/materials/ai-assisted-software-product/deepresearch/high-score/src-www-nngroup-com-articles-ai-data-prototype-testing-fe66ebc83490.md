## TL;DR
利用生成式 AI（特别是 Claude 3.5 Sonnet）结合 Promptframe（提示词线框图）方法，可以快速生成符合特定统计分布（如帕累托分布）和业务逻辑的高保真模拟数据（表格与 SVG 图表）。这能有效解决用户测试中因假数据导致的注意力偏移问题，提升原型测试的真实性和有效性。

## 核心观点
1.  **假数据破坏测试有效性**：在可用性测试中，如果数据看起来不真实或逻辑自洽性差（如总计错误、日期冲突），用户会不自觉地找茬，从而忽略了对核心交互流程的评估。
2.  **Promptframe（提示词线框图）**：这是一种新颖的交付物，介于线框图和 Prompt 之间。它不只画出表格的形状，还定义了填充该表格所需的业务逻辑、约束条件和数据故事。
3.  **上下文决定数据真实感**：AI 生成数据时必须输入详细的业务背景（如面向大型企业的后台 vs 面向小微商家的 App），这决定了数据的数量级、命名规范和复杂度。
4.  **模拟看不见的数据量**：即使界面只显示 50 行数据，生成时也应告诉 AI 背后有 2000 条数据，以便它计算出合理的汇总值（Total）或分页逻辑，而不是仅基于可见数据进行简单求和。
5.  **拒绝随机，拥抱分布**：真实世界的数据很少是均匀分布的。应指示 AI 使用正态分布（如评分）、帕累托分布（二八定律，如销量）来生成更可信的数据。
6.  **图表即叙事**：生成图表时，不要只给数字，要描述数据故事（例如：过去一年呈下降趋势，但受假日促销影响 12 月有激增）。
7.  **SVG 是最佳格式**：对于图表，要求 AI 输出 SVG 代码。相比 PNG，SVG 可以直接导入 Figma/Sketch 进行编辑（修改字体、颜色），保持设计稿的可维护性。
8.  **AI 算数不可靠，需外挂验证**：LLM 的直接数学计算能力较弱，生成财务数据时，应强制要求其使用 Python（Code Interpreter）进行计算，或要求展示计算过程并进行自查。
9.  **工具特长各异**：根据 NN/g 实测，**Claude 3.5 Sonnet** 在遵循复杂指令、生成 SVG 和维持格式方面优于 ChatGPT 4o；但 ChatGPT 4o 在利用 Python 进行精准计算和导出 Excel/CSV 文件方面更强。

## 可落地做法

### 1. 建立数据生成工作流（面向设计师/PM）
*   **定义 Promptframe**：在设计原型的同时，编写一份数据描述文档。
    *   *输入*：目标受众、数据量级、业务规则（如状态为‘已完成’时，进度条必须是 100%）。
    *   *约束*：格式（日期 `DD/MM/YYYY`，金额 `$xx.99`）、唯一性要求。
*   **迭代生成**：
    *   先生成 5-10 条样本数据，人工确认味道对不对。
    *   确认无误后，批量生成所需的 50-100 条数据。

### 2. 图表生成与导入（面向 UI/交互设计师）
*   **去噪处理**：Prompt 中明确要求去除网格线、图例、阴影，只保留核心数据可视化元素，以便融入现有的 UI 设计系统。
*   **异常值标记**：如果在测试某个处理异常的流程，明确指示 AI：让 ‘Sam H.’ 的销售额只有其他人的 20%，并将其柱状图标记为红色。
*   **格式转换**：
    *   将 AI 生成的 SVG 代码保存为 `.svg` 文件。
    *   拖入 Figma，Ungroup 后即可作为矢量图层编辑。

### 3. 数据校验（面向工程/测试）
*   **逻辑自洽性检查**：使用简单的脚本或 Excel 公式检查 AI 生成的数据（如 `单价 * 数量 == 总价`）。
*   **合成数据脱敏**：如果基于真实数据样本进行扩充，务必确保 Prompt 中不包含任何 PII（个人敏感信息），或者利用本地部署的模型进行生成。

## 检查清单：高保真模拟数据生成

在为原型生成数据时，请对照以下清单：

### 基础设定
- [ ] **受众上下文**：是否指定了行业、公司规模、地理位置（决定货币、日期格式）？
- [ ] **数据总量**：是否定义了数据集的总体积（不仅仅是显示的行数）？
- [ ] **排序逻辑**：是否指定了默认的排序字段（如按最近更新排序）？

### 数据规格
- [ ] **唯一性**：哪些列必须唯一（ID、邮箱、SKU）？
- [ ] **枚举值**：状态/类型列是否限制了具体的选项（Open, In Progress, Closed）？
- [ ] **数值范围**：是否设定了合理的最小值和最大值（如心率 60-120）？
- [ ] **分布模型**：是否指定了数据分布规律（正态分布、均匀分布、帕累托分布）？

### 图表专用
- [ ] **尺寸比例**：是否指定了图表的长宽比或具体像素尺寸？
- [ ] **去噪**：是否要求移除多余的装饰元素？
- [ ] **格式**：是否指定输出为 SVG 格式？
- [ ] **关键故事**：是否包含了需要测试验证的异常点或趋势？

### 质量控制
- [ ] **计算验证**：是否要求 AI 使用 Python 或显式步骤验证汇总数据？
- [ ] **无 PII**：是否确认未上传真实的敏感数据？

## 常见坑与对策

| 常见坑 | 表现 | 对策 |
| :--- | :--- | :--- |
| **数学幻觉** | 列表里的分项之和不等于底部的总计（Total）。 | 1. 强制使用 Code Interpreter (Python) 生成。 <br> 2. Prompt 中要求：先计算总和，再生成表格，最后验证一致性。 |
| **数据过于完美** | 所有曲线都平滑上升，所有评分都是 5 星。 | 1. 指定帕累托分布（20% 的产品贡献 80% 的收入）。 <br> 2. 明确要求引入噪音或异常值。 |
| **视觉杂乱** | 生成的图表包含复杂的坐标轴、背景网格、标题，难以修改。 | 1. Prompt 关键词：Minimalist style（极简风格）。 <br> 2. 明确：Do not include gridlines, legends, or background. |
| **格式不可用** | 生成了 PNG 图片，文字模糊且无法修改文案。 | 1. 强制要求输出 SVG 代码。 <br> 2. 对于表格，要求 CSV 或 Markdown 格式，配合 Figma 插件（如 Google Sheets Sync）使用。 |
| **隐私泄露** | 直接粘贴了客户的真实 Excel 表格让 AI 再生成一点类似的。 | 1. **绝对禁止**上传未脱敏的真实数据。 <br> 2. 手动构造 3-5 行假样本作为 Few-Shot 示例给 AI。 |
