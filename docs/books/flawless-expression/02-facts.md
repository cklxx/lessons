# 事实与知识：信息获取、校验与溯源

正确表达的根基是可靠信息。本章把“获取—验证—溯源—更新”串成一条流程，并延续《一人即是一支队伍》的写作约定：少用术语、多给节奏，让读者能沿着同一条航线完成深度检索与筛选。目标是既能跑出 500+ 资料的广度，也能在关键结论上保持可追溯的深度。

## 你将收获什么
- 一套把研究问题写成“可裁决合同”的启动模板（范围/排除项/成功标准）。
- 一条可复现的深度检索流水线：分层获取、带元数据、可去重、可复盘。
- 一个可复制的证据矩阵模板：结论、证据、来源、冲突、置信度一次对齐。
- 一份最低门槛检查清单：避免用过时数据、二手转述和断章取义支撑关键结论。

## 关键工作流（可直接套用）：获取→验证→溯源→更新
把“查资料”当成流水线而不是手艺活。推荐四步闭环：

1. **获取**：多源分层拉取，并为每条资料记录元数据（时间、作者/机构、地区、类型、潜在偏差）。
2. **验证**：抽查与交叉对照（同指标多来源对齐、关键数字回到原文、冲突证据并列）。
3. **溯源**：把每个数字/引语绑定到最小引用单元（原文段落/图表/DOI/公告版本）。
4. **更新**：为易过时信息设复核日期与降级策略（过期就暂停使用或加风险提示）。

## 研究启动：确认问题与成功标准
先用一句话写出研究问题，明确主体、时间、指标与场景，并列出“不在范围内”的内容，避免无限扩张。启动前确定交付物的格式、时间节点和可容忍的风险，例如是否接受 10% 的缺失数据。随后列出需要收集的变量、精度要求与最新时间点，对难以获取的变量提前规划替代指标。最后写下 2–3 个待检验的假设，后续资料按“是否支持假设”来整理与淘汰，能显著减少返工；这和“指挥官先定航线再起航”的原则一致，让检索进入节奏而不是漫游。

## 获取：多源、分层、带元数据
搜集资料时优先一手源，原始实验数据、官方公告、法律条文通常比二手解读更可靠。每个观点至少寻找两类机构的独立来源（学术、政府、行业、媒体）以降低偏见。为每条资料记录时间、作者、适用范围与潜在偏差，并在引用时保留这些元数据，后续审计会更顺畅；把这些字段写进同一张记录表，就能随时回看“航海日志”而不是在文件夹里找碎片。

## 深度检索工作流（可直接套用）
先把问题拆成主体、时间、地点、指标与限制条件，再转化为布尔或短语搜索串。按照渠道分批查询：学术数据库、政府与监管公告、行业与市场报告，以及开源社区的版本说明或讨论。跨语言检索能补足地域差异，必要时用原文关键词回查。对关键论文或报告沿引用链查找支持与反驳，图表则用反搜确认未被断章取义。检索时随手记录“可信度、时效、新鲜度”三项 1–5 分的速评卡，低分资料直接降级为背景，让后续筛选更像流水线而非手工艺。

### 批量搜集与去重
批量抓取摘要或元数据后，先人工抽查 10%–20% 的标题与摘要，确认搜索串足够精准。去重时按照标题、作者、发布日期与 DOI/URL 建指纹，遇到“同报告不同摘要”时保留最完整版本，并给季度或年度更新标注时间与差异。过时、带宣传性或匿名的条目放入“灰名单”，仅作背景参考，避免污染主证据。每轮抽查都要记录命中率与噪声率，这些数字会决定下一轮搜索串是收缩还是放宽。

### 筛选与排序：让好资料先出现
为每条来源记录可信度、新鲜度与相关度并计算加权分，低于阈值的直接降级。把高分来源放在优先队列先精读，低分来源只保留可复核的原始数字。每轮筛选刻意纳入 1–2 条反例或负面案例，并检查来源分布，避免单一国家、机构或商业立场主导，必要时主动补充立场相反的权威来源。做到这一步，你已经拥有了一支“多声部船员”而非单向风向标。

### 500+ 来源批量采集与跨书对齐
若要同时参考另一本书，先用其目录或关键主题拆出 20–30 个子问题，作为标签体系。先批量拉取 100 条资料验证搜索串与评分标准，确认噪声与重复率可接受后，再分批扩充到 500+ 条，每增加 100–150 条就更新评分阈值。为每条资料记录标题、发布时间、作者或机构、类型、地区、链接、摘要、原始语言，以及“参考书章节→本书章节”的映射，并注明是补充还是反驳。使用表格或脚本按三分制自动算总分，低于阈值的进入背景库，同时保证每个子问题至少有 10–20 条高分来源，稀缺主题则记录缺口并启动定向检索。每个批次都保存 CSV/JSON 快照，保留评分与标签，便于回溯筛选逻辑，也便于和合作者共享同一版本的“航海图”。

## 验证：快速审计工作流
整理证据前快速做三项检查：指标是否覆盖目标时间段、样本量是否充足；同一指标在不同来源间是否相互支持，若矛盾要记录差异与解释；发布日期与采集时间是否足够新，避免使用过时数据。对关键数字随机抽查 10%–20%，将原文与二手引用对照，必要时再找反例测试结论的脆弱点。这些抽查结果也要留痕，下一版就能更快复用抽查比例与脚本。

## 溯源：让每个数字都有出处
引用时区分数字和结论的最小引用单元，避免整段复制导致误解。为重要信息保存原文段落、链接、截图或 DOI，确保未来可再检索。若来源更新，记录版本差异与日期，并说明对结论的影响，保持可追踪性。

## 整理与筛选：把原始资料变成可复用资产
将结论、证据、来源、时间与可信度评分整理成证据矩阵并按分数排序，冲突证据并列展示。把“高置信证据”“背景材料”“假设支撑”分层，减少主叙事的噪声，并为每条资料添加主题标签与适用场景（政策、市场或技术）。源文件与摘要建议存入版本库或数据库，用统一命名规则（日期-来源-主题）方便未来复用。完成矩阵后，再写一段“下一站行动”作为尾注，提醒自己何时回查与补充，就像在航海日志上标记下一段航程。

## 示例（可复制）：证据矩阵模板
> 用法：先把“结论”写成一句话命题，再用证据矩阵把支持与反驳并列出来。缺证据的结论先降级为假设，不要直接发布。

```markdown
| claim_id | 结论（命题一句话） | evidence_id | 证据（数字/引语/图表） | 来源（机构/作者/链接/DOI） | 时间点 | 可信度(1-5) | 新鲜度(1-5) | 相关度(1-5) | 冲突/反例 | 备注 |
|---:|---|---:|---|---|---|---:|---:|---:|---|---|
| 1 |  | 1-1 |  |  |  |  |  |  |  |  |
```

## 检查清单（最低门槛）
1. 研究问题是否包含主体/时间/指标/场景，并明确不在范围内的内容？
2. 每个关键结论是否至少有两类独立来源支持（学术/监管/官方公告/行业报告等）？
3. 引用的数字/图表是否回到原文最小引用单元（段落/图表/附录）而非二手转述？
4. 是否记录元数据（时间、作者/机构、地区、类型、潜在偏差）并可被第三方复核？
5. 冲突证据是否并列展示并写出差异原因（口径、样本、时间窗、定义差异）？
6. 是否有抽查比例（例如 10%–20%）与抽查结果留痕，避免“全靠感觉”的可信度评分？
7. 是否保存了来源快照/版本（PDF 版本、公告编号、报告期），避免“今天能打开、明天打不开”？
8. 是否为易过时信息设置复核日期与降级策略（过期就暂停使用或加风险提示）？

## 常见陷阱（失败样本）

### 1) 二手解读当一手证据
- **现象**：引用了一篇“解读文章”的结论，但原文/数据查不到或口径被改变。
- **根因**：省略了回到原文的步骤，把观点转述当成证据。
- **复现**：换一个立场相反的解读来源，得到完全相反结论。
- **修复**：对关键数字与引语，必须回到一手源（论文/公告/原始数据/官方说明）。
- **回归验证**：证据矩阵中每条关键证据都能定位到原文段落/图表编号。

### 2) 版本混用与时间窗漂移
- **现象**：同一指标在不同段落使用不同报告期或不同定义，导致推论不成立。
- **根因**：没有记录版本与时间窗，把“最新数据”当成默认假设。
- **复现**：审阅者按另一时间窗复算，得出相反趋势。
- **修复**：所有指标标注时间窗、报告期与定义；结论只在同口径数据上推导。
- **回归验证**：关键结论的计算过程可被第三方复算，并给出误差区间或敏感性说明。

### 3) 只有支持证据，没有反例策略
- **现象**：证据矩阵只收“支持我”的资料，结论看似牢固但很脆。
- **根因**：把研究当成证明而不是检验，回避反驳与冲突证据。
- **复现**：遇到一个反例场景，结论立刻失效且无法解释。
- **修复**：每个关键结论至少纳入 1–2 条高质量反例/冲突证据，并写清失效条件。
- **回归验证**：结论页附“适用范围/失效条件/需要补证据的空白”，并在更新周期复跑。

## 更新：把知识视为流动资产
为易过时的数据设置复核日期，例如季度财报或政策法规，维护每条信息的信任级别并写清评估理由。当信息过期时预设降级策略：无法及时更新就暂停使用或显式增加风险提示。知识像资产一样需要定期盘点和再平衡。

## 渠道与载体（按类型）
本章刻意避免工具清单，优先把“渠道类型”说清，便于迁移：

- **学术与论文**：优先看原始论文、附录与数据/代码链接；必要时沿引用链找支持与反驳。
- **监管与政府公告**：优先看正式公告、法规条文与解释文件，记录公告编号与生效时间。
- **公司/机构一手材料**：财报、白皮书、产品文档、发布说明；注意宣传倾向与口径变化。
- **行业报告与数据集**：关注采样方法、时间窗、覆盖范围与免责声明。
- **开源社区与版本说明**：版本变更、已知问题与讨论；用于解释“为什么现在不一样了”。
- **更正机制与审计线索**：关注勘误、修订记录与后续澄清；把“更新”当成流程的一部分。
