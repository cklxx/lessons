# 反馈、校对与迭代：把错误变成资产

再精确的初稿也难免遗漏。通过系统化的反馈与校对，可以让表达在每次迭代中更接近“绝对正确”。

## 反馈来源与节奏
反馈可以分层获取。写完立刻自检，再隔一段时间复检，避免熟悉性盲区。同侪审阅最好与目标读者角色相似，看他们能否按文档执行任务。涉及法律、安全、财务等高风险领域时，必须让对应专家审计关键段落。完成执行后记录偏差与返工成本，再反推表达中的缺口，这些数据会成为下一版的改进依据。

## 校对清单（可直接复用）
校对时可以沿着七个问题过一遍：每个数字是否标明时间、单位与来源；关键术语是否在前文或术语表中定义；结论是否与证据一一对应，推理是否写出；适用范围、失效条件与风险是否明确；执行者、截止时间与输出格式是否清楚；是否提供附件、链接或版本号以便审计；如果文档被误解，最可能出现的前三种错误是什么，是否已通过措辞或示例避免。

## 迭代与版本策略
把改动留痕能让审阅更高效。使用语义化版本号（如 0.1 草稿、0.9 候选、1.0 发布），在变更记录中写明修改点与审阅人。配合差异视图，审阅者可以聚焦改动而非重读全文。执行后再把实际结果与假设回填，形成“假设→行动→结果→教训”的闭环，高质量的段落与检查表可以沉淀成模板，持续降低未来的撰写成本。

## 与 AI 协作的注意事项
与模型协作时同样需要结构化提示，明确角色、目标、受众、输出格式和约束，让模型减少自由发挥。要求列出推理链与置信度，并手动校验高风险部分；也可以让模型生成反例、误解清单或“最可能的错误执行方式”，再据此改写原文。这样才能让 AI 成为提高正确性的助力，而不是新的风险源。
