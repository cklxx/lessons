# 结语：让正确性成为默认

绝对正确的表达是一套可习得的系统：识别风险场景、建立事实基线、构造可审计的论证、用无歧义的语言落地，并通过反馈闭环持续迭代。随着流程和模板沉淀，你将逐步降低沟通的熵，使“正确”成为默认输出而非偶然结果。

把本书的思路应用在日常工作中，从一封邮件、一份决策摘要、一次设计评审开始。每次让读者更容易理解、执行和验证，就是在训练自己的“正确性肌肉”。

## 你应该带走的资产（最小集合）
- 一张“正确性分级卡片”：决定哪些沟通必须审计级、哪些可以降级。
- 一张证据矩阵：关键结论的支持与反驳可追溯、可复算。
- 一套论证骨架：结论、证据、推理类型、失效条件可审计。
- 一份歧义扫描规则：把模糊词替换成阈值/窗口/责任人/输出格式。
- 一张错误资产表：把错误变成免疫，而不是重复返工。

## 示例（可复制）：个人“默认正确性”SOP
> 用法：把它当成你写邮件/写 PRD/写报告前的默认流程；必要时再按第 1 章分级提升强度。

```markdown
# 默认正确性 SOP（v1）

## 0) 分级（先决定要做到多正确）
- 场景与后果：<返工/资损/泄露/事故>
- 下游耦合：<个人/团队/跨团队/机器执行>
- 结论：审计级 / 常规级 / 草稿级

## 1) 事实基线（无证据不强断言）
- 关键数字/引语/日期：来源与时间窗写清。
- 冲突证据：至少 1 条，并写出口径差异。

## 2) 论证结构（可审计）
- 结论一句话命题：<命题>
- 证据最小单元：<段落/图表/数据源>
- 推理类型：演绎/归纳/类比/因果
- 失效条件：<什么时候不成立>

## 3) 语言落地（无歧义可执行）
- 模糊词替换：阈值/窗口/条件/责任人/输出格式。
- 行动项：谁在何时交付什么，怎么验收，失败怎么回滚。

## 4) 反馈与回归（把错误变资产）
- 收集失败反馈：读者卡点/误解点/返工成本。
- 入回归：错误资产表 + 检查清单更新。
```
