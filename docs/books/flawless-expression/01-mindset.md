# 心智模型：正确性的定义与边界

绝对正确的表达不是追求形式上的完美，而是让信息在目标场景下被**无歧义理解、无偏差执行、可复现验证**。本章用三个心智模型帮助你判断何时需要追求极高的正确性，并提醒常见的失效点。

## 正确性的三维度
事实、逻辑与语用是并行的三根支柱。事实正确要求信息源可靠、数据可追溯，任何数字、引语、日期都能找到原始出处且可独立验证；逻辑正确强调前提、推论与结论链条自洽，推理步骤应清晰可审计，结论与前提一一对应；语用正确则关注场景与受众，表达需符合角色、语境与约束，避免让读者在缺失信息下自行补全。

## 高风险失败模式
容易让表达偏离正确性的陷阱包括隐含假设（例如默认使用最新数据集或假设用户已登录）、概念漂移（同一术语在不同段落含义变化）、模糊量词（“多数”“尽快”“适当”导致执行偏差）以及证据错位（用经验替代数据，或用过时数据支持当前决策）。写作时刻意扫描这些风险点，往往能提前补上缺口。

## 成本模型：何时值得追求绝对正确
并非所有内容都需要全量校验。法律条款、合规声明、金融报价、医疗或安全指令等高后果场景必须严谨；跨团队协作或自动化执行的高耦合场景也值得额外显式化，因为错误会级联放大；当内容将被程序或代理执行时，更要提供参数化、可复现的描述。根据场景后果和耦合度决定投入多少校验成本，才能避免过度或不足。

## 习惯培养：从“写给自己”到“写给审计者”
预设未来会有第三方审查流程，写作时主动留下可溯源的证据和计算过程；维护术语表，为关键概念写出定义、正反例与边界条件，减少术语漂移；在每次修订时记录日期、原因与影响面，形成版本演进的链路。把读者视作审计者，能自然提高表达的精度。
