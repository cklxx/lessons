# 第 5 章：Computer Use 把“看屏幕”变成工具

Computer use 的核心不是“更会点击”，而是把 UI 操作标准化成模型可理解的工具调用。官方工具文档强调它是一个具备视觉输入和交互输出的通道，模型看到屏幕，再通过一系列受控的动作完成任务。你可以把它理解为“远程操控一台受限的计算机”，而不是让模型随意接管桌面。[15]

Anthropic 的相关博客描述了 computer use 的能力演进，尤其是在新模型和新的交互方式上线后，模型可以更稳定地完成跨页面、多步骤的 UI 任务。这些文章的重点并不是炫技，而是说明这类能力需要与安全边界一起设计，否则“能做”并不等于“适合做”。[46][47]

因此，computer use 更像是最后一公里的补充：当 API 没有覆盖到某些流程，而 UI 已经是系统的真实入口时，它让 agent 有机会完成任务。但它也要求更严格的权限设置和审计策略，因为 UI 通道的作用范围天生更宽。[15][46]
