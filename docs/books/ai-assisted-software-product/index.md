# 如何在 AI 的辅助下完成一个软件产品

> 从需求挖掘、PRD、原型与 UI，到后端架构、Agent/RAG、LLM 预训练/后训练与推理部署的全流程实战。

- **作者**：待补充
- **状态**：大纲版（v0.1）
- **适读人群**：产品经理 / 设计师 / 全栈工程师 / 技术负责人 / 创业者
- **阅读收益**：把 AI 变成“产研设计一体化”的工作台，用可交付物驱动从 0 到 1 的落地

## 本书承诺

你将用一套可复用的流程完成一个真实的软件产品闭环，并且在每个阶段都回答同一个问题：

> **AI 能帮我更快、更稳地产出什么可验收的交付物？我如何验证它是真的对的？**

本书坚持三条主线：

1. **交付物驱动（Artifact-driven）**：每章都有可直接复用的产出（模板、清单、评测集、接口约定）。
2. **评测优先（Evaluation-first）**：在需求、检索、Agent、模型训练与上线阶段，都先定义指标与验收标准。
3. **人类负责边界（Human-in-the-loop）**：AI 负责加速与扩展，人类负责目标、约束、风险与最终决策。

## 贯穿案例（建议跟做）

全书以一个“带检索与工具调用的知识助理 SaaS”为贯穿案例（可替换为你自己的产品）：

- 面向：个人与小团队的知识管理 / 客服 / 内部问答
- 关键模块：文档导入与索引、RAG 问答、任务型 Agent、用户与权限、付费订阅、观测与评测

## 本书交付物清单（读完你将拥有）

- 一份可落地的 **PRD**（含指标、验收标准、非功能性需求、风险清单）
- 一套可点击的 **原型与 UI 规范**（含设计系统与组件清单）
- 一套可运行的 **前后端工程骨架**（含 CI、测试与观测）
- 一条可复用的 **RAG + Agent** 管线（含评测集与回归策略）
- 一个完整的 **用户/权限/付费** 基建（可直接复用到其它产品）
- 一套 **模型训练与部署** 的决策框架（何时训练、如何评估、如何上线）
