# 第 2 章：需求挖掘：从问题到机会

## 本章目标

- 识别“值得做”的问题：痛点强度、频率、付费意愿与替代方案
- 用 AI 加速访谈、归纳与机会评估，但不替代一线事实
- 把需求从“想法”转成可验证的假设与实验

## 章节结构（大纲）

### 2.1 定义问题空间

- 谁在什么场景下遇到什么问题（用户、场景、任务、障碍）
- 框架：JTBD、Opportunity Solution Tree、价值主张画布

### 2.2 访谈与调研：用 AI 做“助理”，不是“结论”

- 访谈提纲生成与迭代：开放问题、追问树、反例提问
- 访谈记录整理：主题聚类、痛点频率、引用证据（原话）
- 常见误区：只问“想不想要”，不问“现有替代”

### 2.3 竞品与替代分析

- 竞品不是对手：替代路径、工作流、成本结构
- AI 辅助：提炼差异点、发现定价与功能缺口、整理对比表

### 2.4 机会评估与优先级

- RICE / ICE / Kano / MoSCoW
- 约束：技术可行性、数据可得性、合规风险、商业化路径

### 2.5 形成可验证假设与实验计划

- 假设三件套：人群 + 场景 + 可量化结果
- 最小实验：landing page、原型测试、Wizard-of-Oz、人工后台

## 本章交付物

- 用户画像与场景清单（含证据）
- 机会树（Opportunity Solution Tree）
- 需求优先级表（含评分依据）
- 实验计划（目标、样本、指标、停止条件）
