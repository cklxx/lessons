# 第 17 章：评测体系：离线/在线、红队与回归

## 本章目标

- 建立可重复、可解释、可回归的评测体系
- 把 RAG/Agent/模型/提示词改动纳入同一套质量门禁
- 用红队与安全评测把风险前置

## 章节结构（大纲）

### 17.1 离线评测：回归集与指标

- 数据结构：问题、参考答案、引用文档、评分规则、标签（难度/主题）
- 指标：正确性、引用一致性、拒答质量、工具调用正确率

### 17.2 线上评测：真实用户与业务指标

- 用户反馈：点赞/踩、原因、是否解决
- 业务指标：留存、转化、工单减少、成本

### 17.3 红队与安全评测

- 注入攻击、越权、数据泄漏、提示词泄漏、工具滥用
- 防护：输入过滤、策略约束、工具沙箱、审计

### 17.4 质量门禁与发布流程

- 在 CI/CD 中加入评测：回归不过不发布
- 版本化：模型/提示词/检索策略/工具清单

## 本章交付物

- 红队用例清单
- 质量门禁策略（通过阈值与回滚条件）
