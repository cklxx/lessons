# 第 5 章：UI 设计：设计系统、交互与可访问性

## 本章目标

- 建立可复用的设计系统（Design Tokens + 组件库）
- 把原型变成工程可落地的 UI 规格（含边界与状态）
- 用 AI 辅助视觉探索与文案，但用一致性与可访问性守住质量

## 章节结构（大纲）

### 5.1 设计系统的最小集

- Token：颜色、字体、间距、圆角、阴影、动效
- 组件：按钮、输入框、表格、对话框、空状态、Toast、Skeleton
- 规范：命名、变体、状态、响应式断点

### 5.2 交互细节与状态设计

- 反馈：加载、成功、失败、撤销、重试
- 防错：输入校验、危险操作二次确认、权限提示

### 5.3 可访问性（A11y）与国际化

- 对比度、键盘可用、焦点管理、ARIA、屏幕阅读
- 文案与术语：一致性、可翻译性、长度适配

### 5.4 设计交付到研发

- 组件清单、页面规格、标注与切图策略
- 设计变更与版本管理（Design → Code 的同步机制）

## 本章交付物

- 设计 Token 与组件清单（含状态与变体）
- UI 规格说明（含可访问性要求）
- 文案与术语表（可复用）
