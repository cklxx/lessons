# 第 7 章：前端实现：组件化、状态与体验指标

## 本章目标

- 把设计系统与原型落到可维护的前端架构
- 把体验指标（LCP/CLS/INP、错误率）纳入开发与上线
- 用 AI 加速组件/表单/状态机实现，但保持一致性与可测试性

## 章节结构（大纲）

### 7.1 前端架构：页面、组件与状态管理

- 目录组织：pages / features / shared / components
- 状态：本地状态、全局状态、服务器状态（缓存与失效）

### 7.2 表单与数据流：从输入到提交

- 校验策略、错误提示、可恢复性
- 乐观更新、重试、幂等键

### 7.3 性能与可观测性

- 核心体验指标：LCP/CLS/INP、首屏、资源体积
- 前端监控：错误、慢请求、用户行为埋点

### 7.4 安全与隐私

- XSS/CSRF、敏感信息处理、权限驱动 UI

## 本章交付物

- 前端模块划分与组件清单
- 关键页面的状态图（空/加载/错误/无权限）
- 前端性能与错误监控指标表
