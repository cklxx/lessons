# 第 4 章：原型与信息架构：用最小成本验证

## 本章目标

- 用可点击原型快速验证关键流程与信息架构
- 用 AI 加速用户流、页面文案与交互边界的探索
- 把原型转化为可实现的 UI 规格与接口需求

## 章节结构（大纲）

### 4.1 信息架构（IA）与导航

- 页面树、功能分组、术语表
- 防止功能堆砌：围绕核心任务设计路径

### 4.2 用户流（User Flow）

- 关键任务：注册 → 导入 → 检索/问答 → 保存/分享 → 升级付费
- 异常流：权限不足、支付失败、导入失败、检索无结果

### 4.3 原型分层：低保真 → 高保真

- 低保真：布局、路径、关键状态（空/加载/错误）
- 高保真：文案、组件、动效、响应式

### 4.4 可用性测试与迭代

- 任务驱动测试、可理解性、可发现性、错误恢复
- 指标：完成率、耗时、错误率、主观满意度

## 本章交付物

- 信息架构图与页面清单
- 用户流图（含异常流）
- 可点击原型与可用性测试记录
