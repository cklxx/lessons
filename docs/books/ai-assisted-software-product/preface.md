# 序言：一人即是一支队伍

> 当边际实现成本被 AI 压缩，竞争力转移到决策力、想象力与验证速度。序言定义“超级个体”路线与执行节奏。[2][4]

## 为什么是现在
- Transformer 结构奠定了可扩展的序列建模基础，而对齐/指令微调让模型更易于被人类以“指令”方式驱动，形成可编程的生产力工具。[2][41]
- 早期退出/自适应推理等技术减少无效计算，使小团队也能负担推理成本。[3][45]
- 云算力与托管服务普及，使个人可以按需租用 GPU 做训练/评测/部署；但必须把预算与能耗约束显式化（尤其在持续迭代时）。[44]

## 超级个体的定义
- **Caller → Designer → Shaper** 三阶段：
  - *Caller*：熟练调用 API，能用 RAG/Agent 拼装可用产品；交付物是 Demo 与测试。[24][29]
  - *Designer*：掌握架构与数据治理，能设计指标与防护栏；交付物是指标表与审计日志。[5][34]
  - *Shaper*：能改写模型行为（SFT/DPO），形成差异化护城河；交付物是权重、基准与成本报告。[40][42]

## 实践节奏
- 每两周产出一个“最小可行叙事”（MVS）：需求证据、原型、上线指标，避免无效迭代。[4]
- 所有改动必须有基线（性能/成本/质量）对比，无数据支撑的“优化”一律退回。[5][6]
- 在版本控制中保存 Notebook、日志与基准，确保复现与审计。

## 如何阅读本书
- 第一篇到第三篇覆盖“能跑、能卖”，第四篇到第五篇覆盖“能演进”。
- 每章末尾的“交付物与验收”可当作检查清单，对照自己的项目逐项补齐。
